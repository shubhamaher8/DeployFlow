DEPLOYFLOW
CI/CD AUTOMATION

Automated Deployment
for 2-Tier Apps

A complete CI/CD pipeline that builds, containerizes, and deploys a Flask + MySQL application on AWS EC2 — triggered automatically on every git push.

🐳

Containerized

Both Flask and MySQL run in isolated Docker containers orchestrated through Docker Compose with health checks and auto-restart.

⚙️

CI/CD Pipeline

Jenkins pipeline clones the repo, builds the Docker image, and deploys with docker compose — fully automated on every push.

☁️

Cloud Deployed

Runs on an AWS EC2 instance with proper security group configuration for SSH, HTTP, Flask, and Jenkins access.

🔄

Two-Tier Architecture

Clean separation between the application layer (Flask) and data layer (MySQL) with environment-based configuration.

🩺

Health Monitoring

Docker health checks continuously monitor both containers and automatically restart them if they become unhealthy.

📦

Persistent Data

MySQL data survives container restarts through Docker volumes, ensuring no data loss during redeployments.

TECH STACK
Python 3.9 Flask MySQL Docker Docker Compose Jenkins AWS EC2 Ubuntu 24.04
HOW THE PIPELINE WORKS
01 Push to GitHub — Developer pushes code to the main branch of the repository.
02 Jenkins Triggered — Jenkins detects the change and starts the pipeline automatically.
03 Clone & Build — Pipeline clones the repo and builds a fresh Docker image for Flask.
04 Deploy — Docker Compose tears down old containers and spins up updated ones.
05 Live — Application is accessible on port 5000, database on 3306, Jenkins on 8080.
