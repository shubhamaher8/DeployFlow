CI/CD Automation

Automated deployment for two-tier applications

A production-focused CI/CD pipeline that builds, containerizes, and deploys a Flask and MySQL application on AWS EC2 with a clean, repeatable delivery workflow.

Project Snapshot
05 Pipeline stages
Build and deploy

Each push can trigger an automated flow from source pull to refreshed containers.

Container-first setup

Application and database services stay isolated, predictable, and easy to maintain.

AWS runtime

The stack is hosted on EC2 with ports configured for application, database, and CI access.

Core Capabilities
01

Containerized

Both Flask and MySQL run in isolated Docker containers orchestrated through Docker Compose with health checks and auto-restart.

02

CI/CD Pipeline

Jenkins clones the repository, builds the application image, and deploys with Docker Compose on every push.

03

Cloud Deployed

Runs on an AWS EC2 instance with proper security group configuration for SSH, HTTP, Flask, and Jenkins access.

04

Two-Tier Architecture

Clean separation between the application layer and data layer with environment-based configuration.

05

Health Monitoring

Docker health checks continuously monitor both containers and automatically restart them if they become unhealthy.

06

Persistent Data

MySQL data survives container restarts through Docker volumes, ensuring no data loss during redeployments.

Technology Stack
Python 3.9 Flask MySQL Docker Docker Compose Jenkins AWS EC2
Pipeline Flow
01 Push to GitHub Developer updates the main branch and starts the delivery cycle.
02 Jenkins Triggered Jenkins detects the change and launches the pipeline automatically.
03 Clone and Build The latest source is pulled and a fresh Flask image is created.
04 Deploy Docker Compose replaces older containers with the updated release.
05 Live Service The application is served on port 5000, MySQL on 3306, and Jenkins on 8080.
